最近在处理理财2.1.2定制项目时，发现Qo的一个比较大的问题：
在Qo中声明的字段信息，但是并没有在对应的DAO实现类的buildCriteria方法中进行该字段的判断，――①
当使用Qo进行查询时，如果qo.set某字段，比如qo.setUserName();
本意是根据该字段进行数据查询，但是因为原因①，就导致在进行数据库查询时，username的查询条件不起作用，出现错误的查询结果

示例Qo:
public class UserQo extends BaseQo{
	private String username;
}	
错误写法:
public class UserDaoImpl extends BaseDao<User,UserQo> implements IUserDao{
	 protected Criteria buildCriteria(Criteria criteria, UserQo qo) {
	    return null;
    }
}
正确写法：
public class UserDaoImpl extends BaseDao<User,UserQo> implements IUserDao{
	 protected Criteria buildCriteria(Criteria criteria, UserQo qo) {
	    if (qo != null) {
			if(qo.getUsername() != null){
				criteria.add(Restrictions.eq("username", qo.getUsername()));
			}
		}
		return criteria;
    }
}

Qo做一些简单的单表查询，如果需要查询哪个字段，在Qo中添加即可
但是，不建议将整个实体类复制到Qo中

ID不需要再Qo中进行声明，因为Qo集成BaseQo,BaseQo中已经加入了ID信息

理财中如果用到已有的Qo进行查询的话，请先确认下DAO实现类的buildCriteria方法中是否加入了相关字段的判断
如果暂时没有用到的话，那万一以后用了呢(手动滑稽)

模糊查询和非模糊查询也要注意
